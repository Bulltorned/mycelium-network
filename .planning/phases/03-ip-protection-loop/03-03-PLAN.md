4. **Instructions:**

   a. `initialize_arbiter_config(ctx: Context<InitArbiterConfig>)` -- Creates ArbiterConfig PDA. Only callable once. Authority = signer (protocol deployer).

   b. `add_arbiter(ctx: Context<ManageArbiter>, arbiter: Pubkey)` -- Adds pubkey to whitelist. Requires authority == config.authority. Check `arbiters.len() < MAX_ARBITERS`.

   c. `remove_arbiter(ctx: Context<ManageArbiter>, arbiter: Pubkey)` -- Removes from whitelist.

   d. `file_dispute(ctx: Context<FileDispute>, evidence_hashes: Vec<[u8; 32]>, similarity_score: u16, match_type: MatchType)`:
      - Validate: `evidence_hashes.len() <= MAX_EVIDENCE_HASHES`.
      - Validate: `similarity_score <= 10_000`.
      - **Read respondent (ip_asset.creator) from raw AccountInfo data** -- since ip_asset is from the Spore program (cross-program account), we CANNOT use `Account<'info, IPAsset>` without adding Spore as a dependency. Instead, manually deserialize the creator field:
        ```rust
        // IPAsset layout: 8 (discriminator) + 32 (original_creator) + 32 (creator)
        // Creator starts at byte offset 40
        let ip_asset_data = ctx.accounts.ip_asset.try_borrow_data()?;
        require!(ip_asset_data.len() >= 72, DrpError::InvalidIPAsset);
        let creator_bytes: [u8; 32] = ip_asset_data[40..72].try_into().map_err(|_| DrpError::InvalidIPAsset)?;
        let respondent = Pubkey::new_from_array(creator_bytes);
        ```
      - Set dispute fields: claimant = signer, respondent = respondent (from deserialization above), stage = DirectResolution, filed_slot = Clock::get().slot, deadline_slot = filed_slot + DISPUTE_DEADLINE_SLOTS.
      - Emit `DisputeFiled` event.

   e. `resolve_dispute(ctx: Context<ResolveDispute>, resolution: Resolution)`:
      - Verify arbiter is in ArbiterConfig whitelist: `require!(arbiter_config.arbiters.contains(&ctx.accounts.arbiter.key()), DrpError::UnauthorizedArbiter)`.
      - Verify dispute is not already resolved: `require!(dispute.resolution.is_none(), DrpError::AlreadyResolved)`.
      - Set dispute.resolution, dispute.arbiter, dispute.resolved_slot, dispute.stage = Resolved.
      - **CPI to Spore update_status:**
        - Determine new_status based on resolution:
          - InFavorOfClaimant -> IPStatus::Suspended (0x02)
          - InFavorOfRespondent -> IPStatus::Active (0x00) -- no change but confirm
          - PartiallyUpheld -> IPStatus::Disputed (0x01)
        - If new_status != Active (i.e., status should change):
          - Derive DRP authority PDA: seeds = [SEED_DRP_AUTHORITY], bump.
          - Build CPI accounts for Spore UpdateStatus: ip_asset + drp_authority_pda (as signer via PDA seeds).
          - Call `mycelium_spore::cpi::update_status(cpi_ctx, new_status)`.
      - Emit `DisputeResolved` event.

**similarity-oracle/image_hasher.py:**

1. Import imagehash, PIL.Image, io.
2. `def compute_phash(image_bytes: bytes) -> str`: Open image from bytes via `Image.open(io.BytesIO(image_bytes))`, return `str(imagehash.phash(img))` (hex string, 16 chars for 64-bit hash).
3. `def compute_dhash(image_bytes: bytes) -> str`: Same but `str(imagehash.dhash(img))`.
4. `def hamming_distance(hash1_hex: str, hash2_hex: str) -> int`: Convert both via `imagehash.hex_to_hash()`, return `hash1 - hash2` (imagehash overloads subtraction as Hamming distance).
5. `def similarity_score(hash1_hex: str, hash2_hex: str) -> float`: `1.0 - (hamming_distance(hash1_hex, hash2_hex) / 64.0)`. Clamp to [0.0, 1.0].

**similarity-oracle/audio_fingerprinter.py:**

1. Import acoustid, os, tempfile, shutil.
2. `def check_fpcalc_available() -> tuple[bool, str | None]`: Check `shutil.which("fpcalc")` or env var `FPCALC_PATH`. Return (available, path).
3. `def compute_fingerprint(audio_bytes: bytes, filename: str = "audio.mp3") -> tuple[float, str]`:
   - Write bytes to temp file (use tempfile.NamedTemporaryFile with delete=False, suffix from filename extension).
   - Call `acoustid.fingerprint_file(temp_path)`.
   - Delete temp file in finally block.
   - Return (duration_seconds, fingerprint_string).
4. `def compare_fingerprints(fp1: str, fp2: str) -> float`:
   - Use acoustid.compare if available, otherwise return 0.0 with a warning.
   - Note: acoustid.compare requires acoustid API key for server comparison. For local comparison, use `chromaprint.decode_fingerprint` if available. If neither works, log warning and return 0.0 -- full audio comparison is best-effort on Windows.

**similarity-oracle/main.py:**

1. FastAPI app with `title="Mycelium Similarity Oracle"`, running on port 8100.
2. Database connection: `DATABASE_URL` from env var, use psycopg2.connect.
3. **GET /health**: Return `{"status": "ok", "fpcalc_available": bool, "fpcalc_path": str|null, "database_connected": bool}`.
4. **POST /similarity/image**: Accept `file: UploadFile`, optional `threshold: int = 10` (Hamming distance).
   - Read file bytes, compute pHash via image_hasher.compute_phash.
   - Query PostgreSQL: `SELECT pubkey, perceptual_hash_hex FROM ip_assets WHERE perceptual_hash_hex IS NOT NULL AND status = 0`.
   - For each row, compute hamming_distance between query pHash and stored hash.
   - Filter to distance <= threshold.
   - Return JSON: `{"query_phash": str, "matches": [{"pubkey": str, "score": float, "matchType": "exact"|"near_duplicate", "distance": int}]}`.
   - Sort matches by score descending.
   - matchType: "exact" if distance == 0, "near_duplicate" if distance <= threshold.

5. **POST /similarity/audio**: Accept `file: UploadFile`.
   - Check fpcalc available. If not, return `{"error": "fpcalc not installed", "matches": []}` with 200 (graceful degradation, not 500).
   - Compute fingerprint via audio_fingerprinter.compute_fingerprint.
   - Query PostgreSQL: `SELECT pubkey, audio_fingerprint FROM ip_assets WHERE audio_fingerprint IS NOT NULL`.
   - Compare fingerprints (best-effort).
   - Return same shape: `{"query_fingerprint": str, "matches": [...]}`.

6. **POST /similarity/exact**: Accept `{"content_hash": str}`.
   - Query PostgreSQL: `SELECT pubkey FROM ip_assets WHERE content_hash = %s`.
   - Return `{"matches": [{"pubkey": str, "score": 1.0, "matchType": "exact"}]}`.

7. Startup event: log port, database connection status, fpcalc availability.

**src/solana-live-adapter.ts update:**

1. Import `checkExactMatch`, `checkImageSimilarity` from `./services/similarity/similarity-client`.
2. Replace `checkSimilarity` method body (lines ~991-1014). Remove the full search scan.
3. New implementation:
   ```typescript
   async checkSimilarity(
     contentHash: string,
     _ipType: IPType
   ): Promise<SimilarityResult> {
     // Layer 1: Exact content hash match via PostgreSQL
     const exactMatches = await checkExactMatch(contentHash);
     if (exactMatches.length > 0) {
       return { matchFound: true, candidates: exactMatches };
     }

     // Layer 2: Perceptual matching requires image/audio data.
     // When called with just a content hash, we can only do exact match.
     // Image/audio similarity is triggered via the MCP tools that pass binary data.
     return { matchFound: false, candidates: [] };
   }
   ```
4. Add a new method for binary similarity (called by MCP tools that have file data):
   ```typescript
   async checkBinarySimilarity(
     fileBuffer: Buffer,
     fileType: "image" | "audio",
     filename: string
   ): Promise<SimilarityResult> {
     let candidates: SimilarityCandidate[];
     if (fileType === "image") {
       candidates = await checkImageSimilarity(fileBuffer);
     } else {
       candidates = await checkAudioSimilarity(fileBuffer, filename);
     }
     return {
       matchFound: candidates.length > 0,
       candidates,
     };
   }
   ```

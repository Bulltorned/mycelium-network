  <action>
    Create `docs/deployment-runbook.md` covering the complete deployment lifecycle. Structure:

    **1. Prerequisites checklist:**
    - [ ] Deployer wallet funded with >= 15 SOL on mainnet-beta
    - [ ] Paid mainnet RPC endpoint configured (Helius/QuickNode)
    - [ ] GitHub repository secrets configured (list each with source)
    - [ ] Squads multisig created and tested (from Plan 03)
    - [ ] All 5 mainnet keypairs generated (from Plan 01)
    - [ ] Anchor build compiles successfully in CI

    **2. First-time deployment procedure (per program):**
    Step 1: Trigger `.github/workflows/mainnet-deploy.yml` via GitHub UI
    Step 2: CI builds verifiable .so via Docker
    Step 3: CI deploys to mainnet buffer account
    Step 4: CI creates Squads proposal for program write
    Step 5: 2-of-3 multisig members approve via app.squads.so
    Step 6: Squads executes the program write on-chain
    Step 7: CI runs `solana-verify verify-from-repo` to confirm bytecode match
    Step 8: Verify program is live: `solana program show <PROGRAM_ID> -u mainnet-beta`

    **3. Deployment order (CRITICAL -- must follow due to cross-program deps):**
    a. Deploy Spore first (no runtime deps on other programs)
    b. Deploy Hypha (depends on Spore program ID for CPI -- already compiled in)
    c. Deploy Rhizome (depends on Spore for royalty config)
    d. Deploy Meridian (depends on Spore for evidence packages)
    e. Deploy DRP last (depends on Spore program ID -- already compiled in; and DRP authority PDA must match what's in Spore)

    **4. Post-deployment verification:**
    - `solana program show <ID> -u mainnet-beta` for each program
    - `solana-verify verify-from-repo --remote <repo-url> <PROGRAM_ID>` for each program
    - Test IP registration on mainnet (small test with real SOL)
    - Verify DRP->Spore CPI works (file test dispute)

    **5. Upgrade procedure (post-initial deployment):**
    - All upgrades go through the same CI/CD workflow
    - Squads proposal is created automatically
    - 2-of-3 approval required
    - Buffer account holds new bytecode until approved

    **6. Rollback procedures:**
    Scenario A -- Deployment failed mid-way:
    - Buffer account exists but program not written: `solana program close <BUFFER_ADDR>` to reclaim SOL
    - No action needed on-chain if program write never executed

    Scenario B -- Program deployed but buggy:
    - Deploy previous version via same CI/CD workflow (revert git tag, re-trigger)
    - Squads proposal + approval + execution for the revert

    Scenario C -- Authority compromised:
    - Immediately propose authority transfer to new multisig via Squads
    - Requires 2-of-3 current members to approve
    - If multisig itself is compromised: programs are immutable until multisig is recovered (this is by design -- no single backdoor)

    **7. Emergency contacts and escalation:**
    - Primary: Aji Pratomo (multisig member 1)
    - Squads support: app.squads.so
    - Solana Foundation validator support (for network-level issues)

    **8. Cost reference:**
    - Link to docs/registration-cost.md for per-registration costs
    - Deployment cost: ~15 SOL for all 5 programs
    - Ongoing: transaction fees only (programs are rent-exempt)
  </action>

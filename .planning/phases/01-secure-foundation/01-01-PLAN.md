**A. Move keypair validation to constructor (SEC-05):**

In `src/solana-live-adapter.ts`, find the constructor. Currently the keypair is loaded lazily or on first use. Move ALL keypair validation to the constructor so the server fails at startup, not on first request.

Replace the lazy loading with eager validation in the constructor:
```typescript
constructor(rpcUrl: string, keypairPath?: string) {
  const resolvedPath = keypairPath ?? resolve(homedir(), "solana-keys", "id.json");

  if (!existsSync(resolvedPath)) {
    throw new Error(
      `Keypair file not found at ${resolvedPath}. ` +
      `Set SOLANA_KEYPAIR_PATH or create the file.`
    );
  }

  let payer: Keypair;
  try {
    const raw = readFileSync(resolvedPath, "utf-8");
    const secretKey = Uint8Array.from(JSON.parse(raw));
    payer = Keypair.fromSecretKey(secretKey);
  } catch (err) {
    throw new Error(
      `Invalid keypair at ${resolvedPath}: ${(err as Error).message}`
    );
  }

  this.payer = payer;
  this.connection = new Connection(rpcUrl, "confirmed");
  console.error(`Payer: ${this.payer.publicKey.toBase58()}`);
}
```

Make sure to:
- Import `existsSync`, `readFileSync` from `fs` and `resolve` from `path` and `homedir` from `os` (add imports if missing)
- Remove any lazy `loadKeypair()` calls that happen on first request
- Keep the `this.payer` assignment BEFORE `this.connection` so the server never accepts connections with an invalid keypair
